 Signal

 (世, p=0.9) It did not rebuild automatically because ecosystem.config.cjs does not make PM2 watch frontend source files;
 it relies on the inner pnpm dev watcher. That watcher chain has two failure modes visible here:

 1. (世, p=0.95) PM2-level watch is explicitly disabled:

 ```js
   {
     name: 'knoxx-frontend',
     cwd: frontendDir,
     script: 'pnpm',
     args: 'dev',
     watch: false,
   }
 ```

 So PM2 will not restart/rebuild on source edits.

 2. (世, p=0.9) The actual frontend automation is nested inside frontend/package.json:

 ```sh
   concurrently ... \
     "vite build --config vite.bridge.config.ts --watch" \
     "vite build --config vite.app-bridge.config.ts --watch" \
     "shadow-cljs watch app"
 ```

 That means automatic rebuild only works if pnpm dev is healthy.

 3. (世, p=0.86) The browser imports the Shadow-wrapped bridge file:

 ```txt
   /cljs/cljs-runtime/module$dist$bridge$knoxx_app_bridge_es.js
 ```

 But the TypeScript fix first rebuilds:

 ```txt
   frontend/dist/bridge/knoxx-app-bridge.es.js
 ```

 Shadow then has to notice/re-wrap that JS file into dist/cljs/cljs-runtime/.... The current setup does not reliably
 force Shadow to recompile when Vite rewrites dist/bridge/knoxx-app-bridge.es.js, so a frontend restart forces the
 missing Shadow re-wrap.

 Evidence

 (世, p=0.96) ecosystem.config.cjs says knoxx-frontend runs pnpm dev and has watch: false.

 (世, p=0.94) frontend/shadow-cljs.edn maps the app bridge through Shadow as a file target:

 ```edn
   "@open-hax/knoxx-app-bridge"
   {:target :file
    :file "dist/bridge/knoxx-app-bridge.es.js"}
 ```

 So there are two build products:

 - Vite output: dist/bridge/knoxx-app-bridge.es.js
 - Shadow-served output: dist/cljs/cljs-runtime/module$dist$bridge$knoxx_app_bridge_es.js

 (世, p=0.88) pm2 logs knoxx-frontend showed the prior frontend process had repeatedly failed with:

 ```txt
   sh: 1: concurrently: not found
 ```

 After your restart, logs showed the expected watchers starting again:

 ```txt
   [APP_BRIDGE] watching for file changes...
   [BRIDGE] watching for file changes...
   [SHADOW] shadow-cljs - watching build :app
 ```

 Frames

 (世, p=0.86) This is a dev-process bug: the comments in ecosystem.config.cjs describe an auto-watch pipeline, but the
 actual chain is brittle and Shadow does not reliably rewrap Vite bridge output.

 (世, p=0.78) The manual restart worked because it restarted both Vite watchers and shadow-cljs watch app, forcing the
 app bridge to be bundled into Shadow’s served runtime again.

 (世, p=0.62) The earlier concurrently: not found log suggests the frontend watcher had previously been dead or
 crash-looping, so even Vite-level auto rebuild may not have been running until you restarted.

 Countermoves

 (己, p=0.9) The fix is not “just enable PM2 watch” blindly; watching all of frontend/src with PM2 would restart the
 whole frontend process on every edit and fight Vite/Shadow watchers.

 (己, p=0.86) The cleaner fix is to make the bridge pipeline explicit: when Vite rebuilds
 dist/bridge/knoxx-app-bridge.es.js, trigger shadow-cljs compile app or restart only the Shadow app watcher.

 (己, p=0.8) Another acceptable fix is splitting knoxx-frontend into separate PM2 apps: knoxx-css-watch,
 knoxx-vite-bridge-watch, knoxx-vite-app-bridge-watch, and knoxx-shadow-app-watch, so failures like concurrently: not
 found are visible per subprocess instead of hidden inside one wrapper.
